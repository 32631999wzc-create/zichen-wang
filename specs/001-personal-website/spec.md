# Feature Specification: 个人网站 (Personal Portfolio Website)

**Feature Directory**: `specs/001-personal-website`

**Created**: 2026-06-16

**Status**: Draft

**Input**: User description: "构建个人网站，我是景观+城市设计师转型AI产品赛道。网站包括首页、作品集、经历阐述、关于我四部分。5个设计项目作品。色彩系统 #edf6f9 主背景 / #ffddd2 点缀 / #83c5be 少许 / #006d77 突出重点。受众：招聘方为主，同行为辅。"

## User Scenarios & Testing

### User Story 1 - 首页品牌展示 (Priority: P1)

招聘方访问网站，第一眼即被专业设计与转型叙事所吸引，产生深入了解的兴趣。

**Why this priority**: 首页是招聘方形成第一印象的唯一机会。必须同时传达设计品位（证明设计背景功底）和产品思维（证明转型方向可信度）。

**Independent Test**: 打开首页，在 5 秒内应感受到：精致的设计品质、清晰的个人定位（景观/城市设计 → AI产品）、明确的浏览引导。

**Acceptance Scenarios**:

1. **Given** 招聘方访问网站首页，**When** 页面加载完成，**Then** 视觉风格体现极简美学，品牌色系统一（#edf6f9 / #ffddd2 / #83c5be / #006d77）
2. **Given** 招聘方浏览首页，**When** 滚动页面，**Then** 清晰展示"转型故事线"：设计背景 → 跨界思考 → AI产品实践 → 未来愿景
3. **Given** 招聘方看到导航，**When** 点击各导航项，**Then** 平滑导航至对应页面
4. **Given** 招聘方在移动设备上访问，**When** 页面渲染，**Then** 布局自适应，所有功能正常可用

---

### User Story 2 - 作品集浏览 (Priority: P1)

招聘方浏览作品集，深入了解候选人的设计能力和项目经验。

**Why this priority**: 作品集是招聘方评估候选人能力最关键的页面。五个项目的呈现方式直接决定专业可信度。

**Independent Test**: 进入作品集页面，可浏览 5 个项目的卡片网格，点击任一项目可查看详情。

**Acceptance Scenarios**:

1. **Given** 招聘方进入作品集页面，**When** 页面加载，**Then** 以网格布局展示所有项目卡片，每张卡片包含缩略图和项目标题
2. **Given** 招聘方点击某个项目卡片，**When** 进入项目详情页，**Then** 展示项目背景、过程、成果、反思四个板块
3. **Given** 招聘方浏览作品集，**When** 使用筛选或分类功能，**Then** 可按项目类型/时间线筛选
4. **Given** 招聘方在移动设备上浏览，**When** 查看作品集，**Then** 网格自适应为单列布局，图片优化加载

---

### User Story 3 - 经历阐述阅读 (Priority: P2)

招聘方阅读经历页，理解从设计到 AI 的完整转型路径和能力积累。

**Why this priority**: 经历页补充作品集无法传达的叙事维度——展示转型的逻辑性和能力迁移的连贯性。

**Independent Test**: 进入经历页面，可按时间线或主题查看经历，内容包含设计实践与 AI 产品探索两个维度的交叉叙述。

**Acceptance Scenarios**:

1. **Given** 招聘方进入经历页面，**When** 滚动浏览，**Then** 以时间线/叙事流方式呈现转型历程
2. **Given** 招聘方查看经历，**When** 阅读每个阶段，**Then** 包含：角色、核心能力、关键成果、与转型的关联性

---

### User Story 4 - 关于我 (Priority: P2)

招聘方了解候选人的全面画像：专业背景、转型动机、个人特质。

**Why this priority**: 关于我页面是招聘方做"人"的判断的页面——决定是否想与这个人共事。

**Independent Test**: 进入关于我页面，可了解个人背景、技能栈、联系方式，所有信息清晰可读。

**Acceptance Scenarios**:

1. **Given** 招聘方进入关于我页面，**When** 浏览内容，**Then** 包含个人简介、核心技能、联系方式
2. **Given** 招聘方想联系候选人，**When** 查看联系方式，**Then** 提供邮箱/社交媒体链接/可选的联系表单
3. **Given** 招聘方在深色环境下浏览，**When** 触发深色模式切换，**Then** 页面配色自动适配

---

### User Story 5 - 内容管理（后台/维护视角） (Priority: P3)

候选人（所有者）方便地更新博客文章和作品集内容。

**Why this priority**: 网站需可持续维护，但 MVP 阶段可先手动管理内容文件。

**Independent Test**: 通过更新 Markdown 文件即可新增博客文章或修改项目信息，无需修改代码。

**Acceptance Scenarios**:

1. **Given** 候选人有新的博客想法，**When** 新建 Markdown 文件放入指定目录，**Then** 博客页面自动展示新文章
2. **Given** 候选人更新作品信息，**When** 修改结构化数据文件，**Then** 作品集页面反映变更

---

### Edge Cases

- 项目图片加载失败时：显示占位图+项目名称文字兜底
- 深色模式切换时：确保所有颜色映射正确，无对比度问题
- 网络慢/离线时：关键内容（个人简介、导航）优先加载，图片懒加载
- 浏览器不支持 CSS 变量/Grid/其他现代特性时：优雅降级，功能不受损但视觉略有简化
- SEO 爬虫访问时：服务端渲染关键内容，确保搜索引擎能抓取到页面结构和文本

## Requirements

### Functional Requirements

- **FR-001**: 网站 MUST 展示至少 4 个独立页面：首页、作品集、经历、关于我
- **FR-002**: 网站 MUST 实现统一的视觉设计系统（色彩调色板、排版、间距）
- **FR-003**: 网站 MUST 支持浅色/深色模式切换
- **FR-004**: 作品集页面 MUST 以网格布局展示所有项目，支持点击查看详情
- **FR-005**: 每个项目详情 MUST 包含：背景、过程、成果、反思四个板块
- **FR-006**: 导航系统 MUST 在所有页面中保持一致，并标识当前所在页面
- **FR-007**: 网站 MUST 在移动端和桌面端均正常显示和工作（响应式设计）
- **FR-008**: 网站 MUST 提供至少一种联系方式（邮箱或联系表单）
- **FR-009**: 博客/内容更新 SHOULD 通过添加 Markdown 文件实现
- **FR-010**: 网站 MUST 针对搜索引擎进行基础优化（meta tags、语义化 HTML、sitemap）

### Key Entities

- **项目 (Project)**: 设计作品的核心数据单元，包含标题、类型、时间、描述、图片集、背景/过程/成果/反思、标签
- **经历条目 (Experience)**: 转型历程中的里程碑，包含时间、角色/标题、组织、描述、技能标签
- **博客文章 (Blog Post)**: 内容输出，包含标题、日期、摘要、正文、标签
- **个人信息 (Profile)**: 个人简介、头像、技能栈、社交媒体链接

## Success Criteria

### Measurable Outcomes

- **SC-001**: 招聘方在 3 秒内理解网站主人的身份定位（设计师转型 AI 产品）
- **SC-002**: 首页首屏内容在 3G 网络下 2 秒内完成加载
- **SC-003**: 作品集页面所有图片懒加载，不影响首屏性能
- **SC-004**: 全站 Lighthouse Performance 评分 ≥ 90，Accessibility ≥ 90
- **SC-005**: 所有页面在 iPhone SE、iPhone 14、iPad、1920×1080 四种视口下均布局正确
- **SC-006**: 浅色/深色模式切换无闪烁、无配色冲突

## Assumptions

- **目标受众**: 招聘方（HR/设计负责人/AI产品团队负责人）为主，占比约 70%；设计/产品同行次之，占比约 30%
- **语言**: 初期以中文为主，预留国际化能力（后续可按需添加英文版）
- **内容来源**: 5 个设计项目的详细内容、图片素材后续由用户提供
- **部署方式**: 采用 Vercel / Netlify 等零运维平台部署，通过 Git 推送自动发布
- **维护频率**: 低频维护——博客月更 1-2 篇，作品集按项目完成节奏更新
- **SEO 需求**: 主要面向中文搜索环境优化（百度、Bing），兼顾 Google
